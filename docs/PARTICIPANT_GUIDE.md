## Exercise 1: Install BMad Method (10 minutes)

### Goal
Set up the BMad framework in your project folder.

### Steps

**1. Create your project folder**
```bash
mkdir bmad-todo-workshop
cd bmad-todo-workshop
```

**2. Install BMad**
```bash
npx bmad-method install
```

You'll see:
```
Welcome to BMad Method!
Let's set up your project...
```

**3. Choose your stack**

When prompted, select:
- Framework: **React**
- Language: **TypeScript**
- Styling: **TailwindCSS**
- Testing: **Vitest** (default)

Use arrow keys and Enter to select.

**4. Wait for installation**

You'll see:
```
✓ Creating .bmad folder...
✓ Installing agents...
✓ Setting up templates...
✓ Configuring prompts...

Installation complete! 🎉
```

**5. Verify installation**

Check that these folders exist:
```bash
ls -la .bmad/
# Should show:
# - agents/
# - prompts/
# - templates/
# - config.json
```

### ✅ Success Criteria

- `.bmad/` folder exists in your project
- Contains `agents/`, `prompts/`, `templates/` subfolders
- No error messages in terminal

### ⚠️ Common Issues

**"npx: command not found"**
- Your Node.js installation may be incomplete
- Ask facilitator for help

**"Permission denied" error**
- Windows: Run terminal as Administrator
- Mac/Linux: Try `sudo npx bmad-method install`

**Installation hangs**
- Check internet connection
- Try: Ctrl+C to cancel, then run again

## Exercise 2: Create Product Requirements (25 minutes)

### Goal
Use the PM (Product Manager) agent to create a PRD for your to-do app.

### Steps

**1. Open VS Code**
```bash
code .
```

**2. Open Copilot Chat**
- Windows: `Ctrl+Shift+I`
- Mac: `Cmd+Shift+I`

**3. Start the PM agent**

Type in Copilot Chat:
```
@bmad /start-work
```

**4. Answer the PM agent's questions**

The agent will ask several questions. Here are **recommended answers**:

**Q: "What's your project name?"**
```
To-Do App
```

**Q: "Describe your project"**
```
A simple task manager where users can add tasks, mark them complete, and delete them.
```

**Q: "What are the core features?"**
```
- Add new tasks
- Mark tasks as complete
- Delete tasks
- View all tasks in a list
```

**Q: "Who are your target users?"**
```
Individual users who need a simple way to track their to-do items
```

**Q: "Any specific requirements?"**
```
Keep it simple - no user accounts or cloud sync for now
```

**Q: "Ready to generate PRD?"**
```
Yes
```

**5. Review the generated PRD**

The agent creates: `docs/prd.md`

Open it and verify it contains:
- Project overview
- Core features list
- User stories ("As a user, I want...")
- Success criteria

### ✅ Success Criteria

- `docs/prd.md` file exists
- Contains at least 3 user stories
- Stories follow format: "As a [role], I want [feature] so that [benefit]"

### 💡 Tips

**Keep answers concise:**
- The PM agent is conversational - short answers speed things up
- If stuck in a loop, type: `/clear` and start over

**Accept defaults:**
- Agent often suggests options - saying "yes" saves time

**Don't over-scope:**
- Stick to the 4 core features listed
- Resist adding login, databases, etc.

### ⚠️ Common Issues

**Agent keeps asking the same question**
- Type: `/clear`
- Restart with: `@bmad /start-work`

**PRD is too short/missing sections**
- Ask agent: "Can you add user stories?"
- Or: "Can you expand the core features section?"

**Agent suggests features you don't want**
- Respond: "No, let's keep it simpler. Just [list features you want]"

## Exercise 3: Design Architecture (25 minutes)

### Goal
Use the Architect agent to create technical design documents.

### Steps

**1. Start the Architect agent**

In Copilot Chat:
```
@bmad /arch
```

**2. Agent reads your PRD**

You'll see:
```
Reading PRD... ✅
Analyzing requirements...
I have some questions about the architecture...
```

**3. Answer architecture questions**

**Q: "How should we manage state?"**
```
Use React Context for global state
```

**Q: "Where should we store tasks?"**
```
Local storage (browser) for now
```

**Q: "Styling approach?"**
```
TailwindCSS utility classes
```

**Q: "Component structure?"**
```
Yes, create separate components for TodoList, TodoItem, and AddTodoForm
```

**Q: "Ready to generate architecture?"**
```
Yes
```

**4. Review generated files**

The agent creates:
- `docs/architecture.md` - Overall architecture
- `docs/adrs/001-react-context.md` - Architecture Decision Record
- `docs/adrs/002-local-storage.md` - Another ADR

**5. Examine the architecture**

Open `docs/architecture.md` and find:
- **Component hierarchy** diagram
- **Data flow** explanation
- **Technology stack** decisions
- **File structure** layout

### ✅ Success Criteria

- `docs/architecture.md` exists with component diagram
- At least 2 ADR files in `docs/adrs/`
- Component structure shows: App → TodoList → TodoItem + AddTodoForm

### 💡 Understanding ADRs

**Architecture Decision Records (ADRs)** document important technical choices.

Each ADR contains:
- **Context**: Why did we need to decide?
- **Decision**: What did we choose?
- **Consequences**: What are the trade-offs?

Example:
```markdown
# ADR 001: Use React Context for State Management

## Context
We need to share todo state between components.

## Decision
Use React Context API instead of Redux.

## Consequences
+ Simpler setup for small app
+ No external dependencies
- Less scalable for large apps
```

### ⚠️ Common Issues

**"Agent can't find PRD"**
- Make sure `docs/prd.md` exists
- Try: `@bmad /arch --reindex`

**Architecture is too complex**
- Ask: "Can you simplify the component structure?"
- Request: "Use fewer components for now"

**Missing ADR files**
- Ask: "Can you create ADRs for the main decisions?"

## Exercise 4: Build Your First Component (30 minutes)

### Goal
Use the Developer agent to generate a working React component.

### Steps

**1. Start a user story**

In Copilot Chat:
```
@bmad /start-story
```

**2. Select a story**

Agent shows your user stories:
```
1. As a user, I want to add tasks so that I can track what I need to do
2. As a user, I want to mark tasks complete so that I can see my progress
3. As a user, I want to delete tasks so that I can remove unwanted items

Which story would you like to implement? (1-3)
```

Type:
```
1
```

**3. Answer implementation questions**

**Q: "What should we name the component?"**
```
AddTodoForm
```

**Q: "Should we validate input?"**
```
Yes, don't allow empty tasks
```

**Q: "What happens after adding a task?"**
```
Clear the input field
```

**Q: "Include tests?"**
```
Yes
```

**4. Agent generates files**

You'll see:
```
Generating component...
✓ Created src/components/AddTodoForm.tsx
✓ Created src/components/AddTodoForm.test.tsx
✓ Updated src/App.tsx

Files ready! Want me to explain the code?
```

Type:
```
Yes, explain
```

**5. Review the generated code**

Open `src/components/AddTodoForm.tsx`

You'll see:
- TypeScript interface for props
- State management with `useState`
- Form submission handler
- Input validation
- TailwindCSS styling

**6. Install dependencies**

```bash
npm install
```

**7. Run the development server**

```bash
npm run dev
```

Open browser to: `http://localhost:5173`

**8. Test your component**

- Type in the input field
- Click "Add Task" button
- Verify the input clears after clicking
- Try adding an empty task (should be prevented)

### ✅ Success Criteria

- `AddTodoForm.tsx` component file exists
- Component renders in the browser
- Can type in the input field
- "Add Task" button is clickable
- Input clears after submission
- Empty tasks are prevented

### 💡 Understanding the Generated Code

**The Developer agent read 3 documents to generate this:**

1. **PRD** → Knew the feature was "add tasks"
2. **Architecture** → Knew to use TypeScript and TailwindCSS
3. **User Story** → Knew validation and input clearing were required

**This is context engineering in action!**

**Key code sections:**

```typescript
// 1. TypeScript interface (from Architecture)
interface AddTodoFormProps {
  onAddTodo: (text: string) => void;
}

// 2. State management (from Architecture: React hooks)
const [text, setText] = useState('');

// 3. Validation (from PRD: prevent empty tasks)
if (text.trim()) {
  onAddTodo(text);
  setText('');  // Clear after add (from user story)
}

// 4. Styling (from Architecture: TailwindCSS)
className="border rounded px-4 py-2 w-full"
```

### ⚠️ Common Issues

**"npm run dev" fails**
- Check Node.js version: `node --version` (need v20+)
- Try: `npm install` again
- Ask facilitator for help

**Component doesn't appear in browser**
- Check `src/App.tsx` - is `<AddTodoForm />` imported?
- Check console for errors (F12 in browser)

**TypeScript errors in VS Code**
- Red squiggles are sometimes false positives
- If app runs in browser, you're okay
- Can ask agent: "Fix TypeScript errors"

**Input field doesn't work**
- Check browser console for errors
- Verify `onChange` handler is present
- Try refreshing the page

## 🎓 What You Learned

### The 4-Phase BMad Workflow

1. **Analysis** (Exercise 2)
   - PM agent captured requirements
   - Created structured PRD document
   
2. **Planning** (Exercise 3)
   - Architect agent designed system
   - Documented decisions in ADRs
   
3. **Solutioning** (Exercise 4)
   - Developer agent planned implementation
   - Created component specifications
   
4. **Implementation** (Exercise 4)
   - Developer agent generated code
   - Code matched PRD + Architecture

### Key Insights

**✅ Context Engineering Wins**
- Documents create shared context between agents
- Each agent builds on previous agent's output
- Better context = better generated code

**✅ Structured Workflow Beats Ad-Hoc Prompting**
- Following BMad phases produces consistent results
- Each phase has a clear purpose
- Agents know their role

**✅ AI Agents Collaborate Through Documents**
- PRD = source of truth for features
- Architecture = source of truth for technical decisions
- User Stories = source of truth for implementation details
