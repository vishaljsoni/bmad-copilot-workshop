## How BMad Works with GitHub Copilot

BMad v6 installs **custom agents** and **slash commands** directly into your project. Once installed, you invoke agents using slash commands in a fresh Copilot Chat session — no file attachments required.

**The pattern is:**

1. Install BMad → slash commands and agent definitions are registered in `_bmad/`
2. Open a **new** Copilot Chat
3. Type the slash command to invoke the agent (e.g. `/bmad-pm`)
4. Run the relevant workflow command (e.g. `/bmad-create-prd`)

## Exercise 1: Install BMad Method (10 minutes)

### Goal
Set up the BMad v6 framework in your project folder.

### Steps

**1. Create your project folder**
```bash
mkdir bmad-todo-workshop
cd bmad-todo-workshop
git init
```

**2. Install BMad v6**
```bash
npx bmad-method install
```

> If you get a stale/old version, use: `npx bmad-method@6.0.1 install`

**3. Follow the installer prompts**

When prompted:
- **Installation location:** Current directory (press Enter)
- **AI Tools:** Choose **GitHub Copilot**
- **Modules:** Select **BMad Method (BMM)**

**4. Wait for installation to complete**

You'll see output similar to:
```
✓ Installing BMad Method...
✓ Compiling agents...
✓ Setting up workflows...

Installation complete! 🎉
```

**5. Verify installation**

Check that these folders exist:
```bash
ls -la
# Should show:
# _bmad/        ← BMad agents, workflows, and configuration
# _bmad-output/ ← Where your generated documents will be saved
```

**6. Test the slash commands**

Open a new Copilot Chat and type:
```
/bmad-help
```

You should get an intelligent guide response listing the available commands and agents.

### ✅ Success Criteria

- `_bmad/` folder exists (note: underscore prefix, NOT `.bmad/`)
- `_bmad-output/` folder exists
- `/bmad-help` responds in Copilot Chat
- No error messages in terminal

### ⚠️ Common Issues

**"npx: command not found"**
- Your Node.js installation may be incomplete
- Ask facilitator for help

**"Permission denied" error**
- Windows: Run terminal as Administrator
- Mac/Linux: Try `sudo npx bmad-method install`

**Installation hangs**
- Check internet connection
- Try: `Ctrl+C` to cancel, then run again

**Slash commands not responding**
- Restart VS Code and verify `_bmad/` folder exists
- Try `/bmad-help` again in a fresh chat

**Q: "Advanced Elicatation"**
```
Advanced Elicitation - Project Discovery Enhancement

I've analyzed our conversation about your intelligent task tracking app. Here are 5 methods that will help us deepen requirements and uncover hidden needs:

Choose a number (1-5), [r] to Reshuffle, [a] List All, or [x] to Proceed:

  - First Principles Analysis - Strip away assumptions about "task management" and rebuild from what you fundamentally need. Why dependencies? Why prediction? What core problem are we really solving?
  - What If Scenarios - Explore edge cases: What if you have 100 asks? What if a dependency chain is 10 levels deep? What if predictions are wrong? What happens when patterns change?
  - Pre-mortem Analysis - Imagine this app fails after 6 months. You stopped using it. Why? What made it annoying or useless? Let's prevent that now.
  - User Persona Focus Group - Even though it's for you, let's explore different "versions of you" - stressed-deadline-you, planning-ahead-you, overwhelmed-you. What does each need?
  - Reverse Engineering - Start from your ideal end state: "I never miss deadlines and always know what to work on next." Work backwards to required features.

[r] Reshuffle for 5 new methods
[a] List all 50 methods
[x] Proceed with current classification

What would you like to explore?
```
Explore various methods or ask it to `suggest your top 3 that are best suited to my application`

**Example: Pre-mortem Analysis**
```
Pre-mortem Analysis: Imagining Failure

It's September 2026 - six months from now. You built the task tracking app, used it for a while, but now it's sitting abandoned. You've gone back to sticky notes, a simple text file, or some other tool.

Let me ask you to imagine this failure scenario:

What happened? Why did you stop using it?

Let me seed some possibilities to get you thinking:

- "Too much friction" - Was it the data entry? Setting up dependencies felt tedious? Progress tracking required too much manual updating?
- "Predictions were annoying" - Did the slip predictions cry wolf too often? Were they not accurate enough? Did constant warnings create alert fatigue?
- "Couldn't trust it" - Did the prediction algorithm feel like a black box? Were you second-guessing its analysis instead of trusting it?
- "Doesn't match my reality" - Did it assume steady progress when your work is actually bursty? Did dependencies feel too rigid for your flexible workflow?
- "Feature creep killed it" - Did you over-build? Too many bells and whistles when you just needed something simple and fast?
- "Maintenance burden" - Did keeping task statuses updated become another todo item? Did old completed tasks clutter everything?

Think about your past experiences with productivity tools. What pattern
typically causes you to abandon them? What would make THIS app join that graveyard?
```

## Exercise 4: Design Architecture (15 minutes)

### Goal
Use the Architect agent to create technical design documents.

### Steps

**1. Start a NEW Copilot Chat session**

> ⚠️ Always start a fresh chat for each new workflow in BMad v6. This prevents context overflow issues.

**2. Invoke the Architect agent**

```
/bmad-architect
```

Then start the architecture workflow:
```
/bmad-create-architecture
```

The agent will automatically read your PRD from `_bmad-output/`.

**3. Answer architecture questions**

**Q: "What I can help you with?"**
```
🏗️ Winston - Architect

Hey user! Winston here. I'm your system architect - I help you design technical
solutions that are pragmatic, scalable, and actually ship. Let's turn your
product vision into a solid technical foundation.

What I can help you with:

  - [MH] Redisplay Menu Help
  - [CH] Chat with the Agent about anything
  - [CA] Create Architecture: Guided Workflow to document technical decisions to
keep implementation on track
  - [IR] Implementation Readiness: Ensure the PRD, UX, and Architecture and Epics
and Stories List are all aligned
  - [PM] Start Party Mode
  - [DA] Dismiss Agent

💡 Tip: You can type /bmad-help at any time to get advice on what to do next.
For example: /bmad-help where should I start with an idea I have that does XYZ

What would you like to do? (Enter a number or command)
```

Answer `CA` to continue and follow the prompts.

**4. Verify the generated architecture**

The architecture document will be saved to `_bmad-output/architecture.md`. It should contain:
- Component hierarchy
- Data flow explanation
- Technology stack decisions
- File structure layout

### ✅ Success Criteria

- `_bmad-output/architecture.md` exists with component diagram
- Component structure shows: App → TodoList → TodoItem + AddTodoForm
- ADRs (Architecture Decision Records) included

### 💡 Understanding Architecture Decision Records (ADRs)

In BMad v6, the Architect agent produces ADRs documenting important technical choices. Each ADR contains:
- **Context**: Why did we need to decide?
- **Decision**: What did we choose?
- **Consequences**: What are the trade-offs?

Example:
```markdown
# ADR 001: Use React Context for State Management

## Context
We need to share todo state between components.

## Decision
Use React Context API instead of Redux.

## Consequences
+ Simpler setup for small app
+ No external dependencies
- Less scalable for large apps
```

### ⚠️ Common Issues

**"Architecture is too complex"**
- In the same chat: "Can you simplify the component structure?"
- Request: "Use fewer components for now"

**"Agent can't find the PRD"**
- Verify `_bmad-output/prd.md` exists from Exercise 2

## Exercise 5: Build Your First Component (30 minutes)

### Goal
Use the Scrum Master (SM) and Developer (DEV) agents to generate a working React component.

### Steps

**1. Start a NEW Copilot Chat and create a story**

```
/bmad-sm
/bmad-sprint-planning
```

Then create the first story:
```
/bmad-create-story
```

Select **Story 1: Add new tasks** and answer `Y` to defaults. The story file will be saved to `_bmad-output/stories/`.

**2. Start a NEW chat and implement the story**

> ⚠️ Open a **new chat session** to implement the story.

```
/bmad-dev-story
```

**3. Answer implementation questions**

**4. Apply the generated code**

The DEV agent will output the code. Create the files:
- `src/components/AddTodoForm.tsx`
- `src/components/AddTodoForm.test.tsx`

**5. Set up the React project (if not already done)**

```bash
npm create vite@latest . -- --template react-ts
npm install
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

**6. Run the development server**

```bash
npm run dev
```

Open browser to: `http://localhost:5173`

**7. Test your component**

- Type in the input field
- Click "Add Task" button
- Verify the input clears after clicking
- Try adding an empty task (should be prevented)

**8. Review the code (optional)**

Open a new chat and run the code review agent:
```
/bmad-code-review
```

### ✅ Success Criteria

- `AddTodoForm.tsx` component file exists
- Component renders in the browser
- Can type in the input field
- "Add Task" button is clickable
- Input clears after submission
- Empty tasks are prevented

### 💡 Understanding the Generated Code

**The Developer agent used 3 documents to generate this:**

1. **PRD** → Knew the feature was "add tasks"
2. **Architecture** → Knew to use TypeScript and TailwindCSS
3. **Story** → Knew validation and input clearing were required

**This is context engineering in action** — each agent builds on the previous agent's output, sharing context through the documents in `_bmad-output/`.

**Key code sections:**

```typescript
// 1. TypeScript interface (from Architecture)
interface AddTodoFormProps {
  onAddTodo: (text: string) => void;
}

// 2. State management (from Architecture: React hooks)
const [text, setText] = useState('');

// 3. Validation (from PRD: prevent empty tasks)
if (text.trim()) {
  onAddTodo(text);
  setText('');  // Clear after add (from story)
}

// 4. Styling (from Architecture: TailwindCSS)
className="border rounded px-4 py-2 w-full"
```

### ⚠️ Common Issues

**"npm run dev" fails**
- Check Node.js version: `node --version` (need v20+)
- Try: `npm install` again
- Ask facilitator for help

**Component doesn't appear in browser**
- Check `src/App.tsx` — is `<AddTodoForm />` imported?
- Check console for errors (F12 in browser)

**TypeScript errors in VS Code**
- Red squiggles are sometimes false positives
- If app runs in browser, you're okay

## 🎓 What You Learned

### The BMad v6 Workflow with GitHub Copilot

| Phase | Agent | Slash Commands | Output |
|-------|-------|----------------|--------|
| **Analysis** | PM | `/bmad-pm` → `/bmad-create-prd` | `_bmad-output/prd.md` |
| **Planning** | PM | `/bmad-pm` → `/bmad-create-epics-and-stories` | Epic and story files |
| **Architecture** | Architect | `/bmad-architect` → `/bmad-create-architecture` | `_bmad-output/architecture.md` |
| **Story Prep** | SM | `/bmad-sm` → `/bmad-sprint-planning` → `/bmad-create-story` | Story file |
| **Implementation** | DEV | `/bmad-dev-story` | Component files + tests |
| **Review** | DEV | `/bmad-code-review` | Code review feedback |

### Key Insights

**✅ Slash Commands for Every Agent**
- `/bmad-pm` — invoke the Product Manager agent
- `/bmad-architect` — invoke the Architect agent
- `/bmad-sm` — invoke the Scrum Master agent
- `/bmad-dev-story` — invoke the Developer agent to implement a story
- `/bmad-help` — your intelligent guide at any time

**✅ Shorthand Responses Save Time**
- `Y` — yes / agree
- `N` — no / decline
- `A` — accept **all** remaining defaults
- Always start fresh: open a new chat for each workflow

**✅ Context Engineering Wins**
- Documents create shared context between agents
- Each agent builds on previous agent's output
- Better context = better generated code

**✅ BMad v6 Folder Structure**
```
your-project/
├── _bmad/              ← BMad config, agents, and workflows (don't edit)
│   └── bmm/
│       ├── agents/     ← Agent definitions
│       ├── workflows/  ← Workflow definitions
│       └── tasks/      ← Task definitions
├── _bmad-output/       ← Your generated artifacts
│   ├── prd.md
│   ├── architecture.md
│   └── stories/
└── src/                ← Your actual code
```
